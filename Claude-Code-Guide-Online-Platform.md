## 1. Как это работает — вы говорите, Claude делает

### Что такое Claude Code

Claude Code — это AI-разработчик внутри приложения Claude на вашем Mac или PC. Вы пишете задачу обычным языком, Claude выполняет: создает сайт, подключает оплату, настраивает сервер, исправляет ошибки.

Вам не нужно знать программирование. Не нужно видеть код. Не нужно понимать команды. Вы работаете в том же приложении Claude, в котором общаетесь с AI — просто в режиме "Code".

### Как выглядит рабочий процесс

1. Вы открываете приложение Claude на компьютере
2. Переключаетесь в режим Code (раздел "Code" в боковом меню)
3. Выбираете папку проекта на компьютере
4. Пишете задачу обычным языком
5. Claude выполняет и дает ссылку для проверки
6. Вы открываете ссылку в браузере, смотрите результат
7. Если что-то не так — говорите, Claude исправляет

### Пример

Вы пишете:
> "Создай страницу с каталогом моих курсов. Три колонки. Карточка: название, картинка, цена, кнопка Подробнее. Дизайн современный, минималистичный."

Claude:
- Создает файлы (вы их не видите и не трогаете)
- Запускает сайт локально
- Дает вам ссылку: `http://localhost:3000/courses`
- Вы открываете в браузере и видите готовую страницу

Если не нравится — пишете: "Сделай карточки крупнее, добавь рейтинг звездами." Claude тут же исправляет.

### Почему это работает

Claude Code — не чат-бот, который отвечает на вопросы. Это агент, который действует на вашем компьютере: создает файлы, устанавливает программы, подключается к серверам, тестирует результат в браузере. Вы — заказчик и проверяющий. Claude — исполнитель.

## 3. Начало работы: пошагово от скачивания до первой задачи

### Шаг 1: Зарегистрируйтесь на claude.ai

Перейдите на сайт claude.ai, создайте аккаунт, оформите подписку Pro или Max.

### Шаг 2: Скачайте приложение Claude

На странице claude.ai/download скачайте приложение для Mac или Windows. Установите как обычную программу — перетащите в "Программы" (Mac) или запустите установщик (Windows).

### Шаг 3: Создайте папку для проекта

Откройте Finder (Mac) или Проводник (Windows). Создайте новую папку в удобном месте:
- Mac: `Документы/my-platform`
- Windows: `Документы/my-platform`

Название папки — латиницей, без пробелов. Это будет "дом" вашего проекта. Все файлы сайта Claude будет создавать внутри этой папки.

### Шаг 4: Откройте Claude Code и привяжите папку

1. Откройте приложение Claude
2. Перейдите в раздел **Code** (в боковом меню приложения)
3. Нажмите **"Select folder"** (или "Выбрать папку")
4. Укажите папку, которую создали на шаге 3

Claude Code теперь привязан к этой папке. Все что он создает — появляется внутри неё. Все что он ищет — ищет внутри неё. Это его рабочая территория.

### Шаг 5: Разрешения — Claude будет спрашивать "можно?"

При первой работе Claude будет спрашивать разрешение на действия: "Можно создать файл?", "Можно выполнить команду?", "Можно установить пакет?"

**Что делать:**
- В начале нажимайте **"Allow"** (Разрешить) на каждый запрос — так вы поймете что Claude делает
- Когда привыкнете, можно включить режим **"Auto-allow"** (Авторазрешение) — Claude не будет спрашивать каждый раз
- Если вы не уверены — нажмите **"Deny"** (Отклонить). Claude предложит альтернативу

Разрешения — это защита. Claude не может сделать ничего критичного без вашего одобрения.

### Шаг 6: Первая задача — настройка инструментов

Напишите Claude:

> "Настрой все для начала работы над проектом. Установи нужные инструменты для разработки. Настрой Git и GitHub с моим именем [Ваше Имя] и email [ваш@email.com]. Создай репозиторий для проекта."

**Что произойдет:**
- Claude установит инструменты разработки (Node.js, Python и др.)
- Настроит Git (систему версий)
- Попросит вас зарегистрироваться на GitHub (если еще не зарегистрированы)
- Создаст SSH-ключ (цифровой пропуск для GitHub) и покажет куда его вставить на сайте GitHub
- Создаст репозиторий — облачную копию вашего проекта

Claude будет делать это пошагово и объяснять если что-то нужно от вас. Просто следуйте его инструкциям.

### Шаг 7: Проверьте что все работает

Напишите: "Проверь что все настроено правильно."

Claude проведет диагностику и скажет если что-то не так.

**С этого момента все технические задачи вы решаете через Claude.** Просто описывайте что нужно.

## 4. Ежедневная работа: сессии, разрешения, модели

### Как начать новую рабочую сессию

1. Откройте приложение Claude
2. Перейдите в раздел Code
3. Выберите свою папку проекта (если не выбрана)
4. Начните новый разговор или продолжите предыдущий

**Claude забывает все между сессиями** — но читает файл CLAUDE.md (раздел 6), который хранит правила и контекст проекта. Поэтому CLAUDE.md так важен.

**Чтобы продолжить вчерашнюю работу**, можно:
- Открыть предыдущую сессию из истории в приложении
- Или начать новую и сказать: "Продолжаем работу над [задачей]. Посмотри что уже сделано и продолжай."

### Модели: Opus и Sonnet — когда что использовать

В Claude Code доступны две модели — как два режима работы:

| Модель | Что умеет лучше | Скорость | Расход лимита |
|--------|----------------|----------|---------------|
| **Opus** | Сложные задачи: архитектура, дебаг запутанных багов, новые фичи с нуля | Медленнее | Тратит больше |
| **Sonnet** | Простые задачи: правки текста, мелкие фиксы, работа по готовому плану | Быстрее | Тратит меньше |

**Как переключить:** напишите `/model sonnet` или `/model opus` в чат.

**Правило:** начинайте с Opus (по умолчанию). Когда задача простая — переключайтесь на Sonnet. Это как ехать на первой передаче в гору и на пятой по шоссе.

### Когда контекст заканчивается

У Claude есть "окно контекста" — объем информации, которую он помнит в рамках одной сессии. Когда оно заполняется, Claude начинает забывать начало разговора.

**Признаки:** Claude повторяется, забывает что делал раньше, путает файлы.

**Что делать:**
- Напишите `/compact` — Claude сожмет контекст, сохранив самое важное
- Или начните новую сессию и скажите: "Посмотри текущее состояние проекта и продолжай работу над [задачей]."

### Что делать если Claude ошибся или сломал что-то

**Claude ошибается — это нормально.** Он не идеален. Вот что делать:

- **Мелкая ошибка** (не тот цвет, текст не так): "Измени [что конкретно] на [как должно быть]."
- **Что-то сломалось после изменений**: "После последних изменений [что именно] перестало работать. Откати и исправь."
- **Все совсем плохо**: "Откати последний коммит" — Claude вернет проект к предыдущему сохраненному состоянию.

Именно поэтому важно регулярно коммитить (сохранять) — это ваши точки отката.

### Сколько стоит сессия

Напишите `/cost` — Claude покажет сколько потрачено в текущей сессии. Это поможет контролировать расход лимита подписки.

## 5. Проект — что это и из чего состоит

### Что такое проект

Проект — это набор файлов на вашем компьютере, из которых собирается сайт. Вы эти файлы не открываете и не редактируете — это делает Claude. Для вас проект — это папка, которую вы указали Claude Code.

### Из чего состоит обучающая платформа

Представьте платформу как здание с отделами:

**Витрина (то, что видят пользователи):**
- Главная страница — описание, тарифы, отзывы, кнопка регистрации
- Каталог курсов — список с фильтрами и поиском
- Страница курса — описание, программа, превью уроков
- Урок — видео, текст, задания
- Личный кабинет — прогресс, подписка, настройки

**Подсобка (невидимая часть):**
- Сервер — обрабатывает запросы, проверяет подписку
- База данных — хранит пользователей, курсы, платежи
- Авторизация — регистрация, вход, восстановление пароля
- Платежи — прием денег, управление подписками

**Админка (только для вас):**
- Создание и редактирование курсов
- Управление пользователями
- Аналитика — сколько учеников, какие курсы популярны, доход

Все это Claude создаст по вашим задачам. Вам не нужно знать как это устроено технически — достаточно описать что должно быть.

### Какие технологии использует Claude

Вам не нужно это запоминать, но полезно понимать что за словами:

- **Next.js** — инструмент для создания сайтов. Как конструктор, из которого собирается визуальная часть
- **FastAPI** — инструмент для серверной части. Обрабатывает данные, подписки, платежи
- **PostgreSQL** — база данных. Как Excel-таблица, но мощнее: хранит пользователей, курсы, прогресс
- **TypeScript** — язык программирования с защитой от ошибок

Claude сам выберет подходящий стек и объяснит почему, если спросите.

## 6. GitHub — облачное хранилище проекта

### Зачем нужен GitHub

GitHub — это облако для вашего проекта. Как iCloud для фотографий, только для файлов проекта.

**Три главных причины:**
1. **Бэкап.** Если сломается компьютер — проект не потеряется. Все в облаке.
2. **История.** Каждое изменение сохраняется. Можно откатить проект к любому моменту. Как машина времени для сайта.
3. **Деплой.** Когда вы публикуете сайт, хостинг берет код прямо из GitHub. Обновили код — сайт обновился автоматически.

### Что от вас нужно

1. Зарегистрироваться на github.com (бесплатно)
2. В первой сессии сказать Claude: "Настрой GitHub для проекта" — он сделает все остальное

### Что полезно понимать

- **"Закоммить"** — сохранить текущее состояние проекта. Говорите Claude: "Закоммить изменения" после каждой завершенной задачи.
- **"Запушить"** — отправить сохраненное в облако (GitHub). Говорите: "Запуш на гитхаб."
- **Репозиторий** — ваш проект на GitHub. Создается один раз в начале.

Все это Claude делает по вашей просьбе. Вы не работаете с GitHub напрямую.

## 7. CLAUDE.md — память и правила Claude

### Проблема: Claude забывает все между сессиями

Каждый раз когда вы открываете новый разговор, Claude начинает с чистого листа. Он не помнит что делал вчера, какие решения принимали, какая структура у проекта.

### Решение: файл CLAUDE.md

**CLAUDE.md** — это файл с инструкциями, который лежит в папке проекта. Claude читает его автоматически в начале каждой сессии. Это его "должностная инструкция" и "память о проекте".

### Что в нем писать

- Что это за проект и для кого
- На каком языке общаться
- Что можно и нельзя делать (например, не деплоить без разрешения)
- Доступы к серверу и базе данных
- Правила и стандарты, которые вы хотите соблюдать

### Как создать

Скажите Claude: "Создай CLAUDE.md для проекта обучающей платформы [название]. Целевая аудитория — [кто]. Функции: курсы, подписка, личный кабинет."

Claude создаст файл. В Части 2 этого документа есть готовый шаблон, который можно скопировать.

### Дополнительная память

Кроме CLAUDE.md, Claude может хранить заметки о проекте в специальной папке memory/. Туда попадают уроки из ошибок, информация об инфраструктуре, доступы. Это происходит автоматически — вам не нужно это настраивать.

### Передача контекста между сессиями

Если задача большая и не помещается в одну сессию, скажите Claude: "Подготовь описание текущего состояния для следующей сессии." Он создаст краткое резюме, которое можно вставить в начало следующего разговора.

## 8. MCP — плагины, расширяющие возможности

### Что такое MCP

MCP (Model Context Protocol) — это плагины для Claude Code. Каждый плагин дает Claude новую способность: работать с браузером, искать в интернете, управлять Google Таблицами.

Без плагинов Claude может писать код и работать с файлами. С плагинами — может еще и тестировать сайт, искать документацию, подключаться к внешним сервисам.

### Какие плагины подключить

| Плагин | Что дает Claude | Когда нужен | Как подключить |
|--------|----------------|-------------|---------------|
| **Playwright** | Открывать сайт в браузере, делать скриншоты, тестировать кнопки | С первого дня | "Подключи браузер для тестирования" |
| **Brave Search** | Искать в интернете: документацию, решения ошибок, примеры | С первого дня | "Подключи поиск в интернете" |
| **Notion** | Работать с вашими задачами и заметками в Notion | Если используете Notion | "Подключи Notion" |
| **Sentry** | Отслеживать ошибки на работающем сайте | После публикации | "Подключи мониторинг ошибок" |
| **Google Workspace** | Работать с Google Таблицами, Документами, Gmail | Если нужны Google-сервисы | "Подключи Google Sheets" |

### Как подключать

Просто скажите Claude нужную фразу из таблицы. Он установит плагин сам. Если понадобится API-ключ (пароль для доступа к сервису) — Claude объяснит где его получить.

## 9. Скиллы — готовые сценарии для Claude

### Что такое скилл

Скилл — это сохраненная инструкция для Claude, которую вы вызываете одной командой. Вместо того чтобы каждый раз объяснять процесс деплоя из 10 шагов, вы говорите `/deploy` — и Claude выполняет все по инструкции.

### Зачем нужны

- **Экономия времени** — не повторять одно и то же каждый раз
- **Стабильность** — Claude выполняет задачу одинаково и не пропускает шаги
- **Безопасность** — в скилле прописаны ограничения (например, не деплоить без подтверждения)

### Какие скиллы создать для платформы

Попросите Claude в начале проекта:

| Скажите Claude | Что он создаст | Как потом вызывать |
|----------------|---------------|-------------------|
| "Создай скилл для деплоя на сервер. Не деплоить без моего подтверждения." | Сценарий публикации сайта | `/deploy` |
| "Создай скилл для выполнения ТЗ по шагам с отчетом после каждого этапа." | Сценарий работы по задаче | `/spec` |
| "Создай скилл для проверки статуса сайта — работает ли, нет ли ошибок." | Сценарий мониторинга | `/status` |
| "Создай скилл для бэкапа базы данных." | Сценарий резервного копирования | `/backup` |

### Встроенные скиллы (уже есть в Claude Code)

- `/simplify` — проверка качества кода
- `/debug` — поиск и исправление ошибки
- `/batch` — одновременные изменения в нескольких местах

## 10. Домен и хостинг — где будет жить сайт

### Домен — адрес сайта

Домен — это адрес, который люди набирают в браузере: mojaplatforma.ru или myplatform.com.

**Где купить:**

| Если аудитория | Зона | Где покупать | Цена |
|----------------|------|-------------|------|
| В России | .ru | reg.ru, beget.com | ~500-1000 руб/год |
| Международная | .com | namecheap.com, cloudflare.com | ~$10-15/год |

Покупается за 5 минут на сайте регистратора. Выберите короткое, запоминающееся имя.

### Хостинг — компьютер для сайта

Хостинг — это компьютер (сервер), который работает 24/7 и отдает ваш сайт посетителям. Без него сайт доступен только на вашем компьютере.

**Два пути:**

| Путь | Что это | Для кого | Цена | Как настроить |
|------|---------|----------|------|---------------|
| **Автоматический** (Vercel + Railway) | Облачные сервисы, все настраивается за вас | Новичкам | $5-20/мес | Claude подключит через GitHub |
| **Свой сервер** (VPS) | Арендованный компьютер, полный контроль | Когда нужно больше контроля | $5-15/мес | Claude настроит полностью |

**Рекомендация:** начните с автоматического пути (Vercel + Railway). Это проще и быстрее. Claude подключит ваш GitHub-репозиторий, и сайт будет обновляться автоматически при каждом изменении.

### Как дать Claude доступ к серверу (если выбрали VPS)

Скажите: "Я купил сервер на [хостере]. IP: [адрес], пароль: [пароль]. Настрой подключение и безопасность."

Claude сам подключится, настроит безопасность, установит все необходимое. Вам не нужно подключаться к серверу самостоятельно.

## 12. Публикация сайта в интернет

### Что такое деплой

Деплой — это момент, когда ваш сайт становится доступен всем в интернете по адресу вашего домена. До этого сайт работает только на вашем компьютере (для проверки).

### Как это происходит

1. Вы проверили что все работает локально
2. Вы говорите Claude: "Задеплой проект"
3. Claude спрашивает подтверждение (это важно — он не деплоит сам)
4. Вы подтверждаете: "Да, деплой"
5. Claude публикует код, подключает домен, настраивает SSL (шифрование)
6. Через 5-15 минут сайт доступен по вашему адресу

### Что такое SSL

SSL — это шифрование. Когда подключен SSL, в браузере отображается замок рядом с адресом сайта (https://). Без SSL браузеры показывают предупреждение "Сайт небезопасен". SSL бесплатен (Let's Encrypt) — Claude подключит автоматически.

### Что такое DNS

DNS — это телефонная книга интернета. Она связывает ваш домен (mojaplatforma.ru) с адресом сервера (123.45.67.89). Claude настроит, но вам нужно будет зайти в панель управления домена (reg.ru или где купили) и добавить записи, которые Claude укажет.

## 13. После запуска — что нужно контролировать

### Бэкапы — страховка от потери данных

Бэкап — это копия базы данных. Если что-то сломается, данные можно восстановить из бэкапа. Попросите Claude настроить автоматические ежедневные бэкапы при деплое.

**Без бэкапов вопрос не "если", а "когда" вы потеряете данные.**

### Мониторинг — знать что сайт работает

- **UptimeRobot** (бесплатно) — проверяет сайт каждые 5 минут. Если упал — пришлет уведомление на email или в Telegram
- **Sentry** (бесплатный план) — отслеживает ошибки в коде. Вы узнаете о проблеме раньше, чем пользователи

Скажите Claude: "Настрой мониторинг: UptimeRobot для доступности, Sentry для ошибок."

### Аналитика — понимать что происходит

Подключите Яндекс.Метрику (для российской аудитории) или Google Analytics (для международной). Скажите Claude — он встроит код аналитики в сайт.

### Если сайт упал

Откройте Claude Code и скажите: "Сайт не работает. Проверь что случилось и исправь." Claude подключится к серверу, найдет причину и устранит.

## 16. Типичные ошибки и как их избежать

### 1. Начинать без CLAUDE.md
Claude без инструкций — как нанятый сотрудник без должностной инструкции. Первым делом создайте CLAUDE.md (готовый шаблон — в Части 2).

### 2. Давать слишком большие задачи
"Сделай всю платформу" — Claude запутается или сделает не то. Разбивайте: сначала регистрацию, потом каталог, потом оплату. По одной функции за раз.

### 3. Не проверять результат
Claude сделал — обязательно откройте в браузере. Покликайте. Попробуйте зарегистрироваться. Нашли проблему — скажите. Это нормальный процесс, не ошибка Claude.

### 4. Не сохранять прогресс
После каждой завершенной задачи говорите: "Закоммить и запуш." Это ваша страховка. Если что-то пойдет не так — можно откатиться.

### 5. Пытаться делать техническую работу самому
Вы не программист — и не должны быть. Не гуглите команды, не пытайтесь редактировать код. Если что-то нужно — скажите Claude. Если он ошибся — скажите в чем проблема, а не пытайтесь исправить сами.

### 6. Забывать про бэкапы
Данные пользователей — это актив вашего бизнеса. Попросите Claude настроить автоматические бэкапы до запуска.

### 7. Деплоить без проверки
Перед публикацией всегда проверяйте локально: все страницы, регистрацию, оплату. Скажите Claude: "Проверь что все работает перед деплоем."
